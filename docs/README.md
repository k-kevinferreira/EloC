# Docs

Centraliza a documentacao do projeto para reduzir dispersao de conhecimento entre frontend, backend, banco e arquitetura.

## Documentos principais

- `codex-guide.md`: guia operacional permanente para uso do Codex no projeto
- `technical/database-model.md`: modelagem atual do banco de dados e decisoes de dominio
- `architecture/README.md`: visao arquitetural e estado atual da estrutura

## Ordem recomendada para retomada

1. ler `codex-guide.md`
2. revisar `technical/database-model.md`
3. revisar `README.md` da raiz
4. revisar `apps/backend/README.md`
